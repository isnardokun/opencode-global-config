Use the full feature workflow for: $ARGUMENTS

Execute in order:

**Phase 1 — @architect:**
Analyze current project structure. Identify files that will be affected. List risks and dependencies. Do not modify anything.

**Phase 2 — @planner:**
Create a phased implementation plan with explicit success criteria for each phase. Include test requirements.

**Phase 3 — @builder with safe-implementation and test-first skills:**
Implement the feature following the plan. Write tests before or alongside code. Make surgical changes only.

**Phase 4 — @reviewer with precommit-review:**
Review all changes made. Flag any blockers. Confirm tests are present. Give final recommendation.
