Antes de modificar código:

1. Leer árbol del proyecto.
2. Detectar stack.
3. Identificar entrypoints.
4. Identificar archivos de configuración.
5. Detectar comandos:
   - test
   - lint
   - build
   - dev
6. Identificar riesgos.

Salida:
- Stack detectado
- Estructura relevante
- Archivos críticos
- Comandos disponibles
- Riesgos iniciales
